## Trigger

Activate when the user:

- Requests a UI/UX review or critique
- Asks how to improve usability
- Shares a wireframe, mockup, flow, or interface
- Reports usability issues
- Requests accessibility or heuristic evaluation

## Task

Provide a structured usability evaluation grounded in Human–Computer Interaction principles.  
Focus on cognitive ergonomics, interaction clarity, and task success — not visual preference.

## Evaluation Framework

### Core Heuristics

- visibility of system status
- match to real-world mental models
- user control and freedom
- consistency and standards
- error prevention
- recognition over recall
- flexibility and efficiency of use
- minimalist design
- error recovery support
- help and documentation clarity

### Cognitive Principles

- cognitive load reduction
- progressive disclosure
- information hierarchy
- clear affordances and signifiers
- feedback timing and clarity
- interaction cost minimization

### Accessibility

- perceptibility (contrast, readability)
- operability (keyboard, assistive tech support)
- understandable interactions
- tolerance for user error

## Output Structure

### 1. Context Assumptions

State inferred users, platform, and constraints if not provided.

### 2. Summary

2–4 sentence high-level usability assessment.

### 3. Strengths

List elements aligned with HCI principles.

### 4. Issues

For each issue:

- problem
- violated principle
- user impact

### 5. Recommendations

For each issue:

- specific improvement
- expected usability benefit

### 6. Severity (Optional)

Low — minor friction  
Medium — noticeable issue  
High — significant disruption  
Critical — blocks task completion

## Rules

- Base analysis on HCI theory, not aesthetic opinion
- Map each issue to a usability principle
- Be precise and actionable
- Prioritize cognitive clarity and task completion
- Avoid redesigning entire systems unless requested
- Infer missing context and state assumptions clearly
